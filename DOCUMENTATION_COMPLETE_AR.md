# التقرير الفني الشامل لمنصة أحلام - مراحل التطوير والتشغيل

تم تطوير منصة أحلام كمنصة ذكية متكاملة لإدارة الاختبارات والتقييم، عبر خمس مراحل تقنية منظمة، تلتها المرحلة السادسة للتوثيق. يوضح هذا التقرير كافة التفاصيل التقنية، المكتبات المستخدمة، وكيفية تفعيل وتشغيل المنصة بكامل ميزاتها.

---

## 1. ملخص مراحل التطوير (1 - 5)

### **المرحلة الأولى: نظام الأدوار والصلاحيات (RBAC) والدعوات**
- **الهدف:** بناء هيكل تنظيمي يدعم تعدد المنظمات وصلاحيات الوصول.
- **ما تم إنجازه:**
    - تحديث جداول قاعدة البيانات لدعم أدوار: **أدمن عام، محاضر (صاحب عمل)، مساعد، ومشترك**.
    - إنشاء نظام دعوات عبر البريد الإلكتروني (بمفاتيح فريدة وتتبع للحالة: مرسلة، مقبولة، مرفوضة).
    - تطبيق سياسات أمان (RLS) تضمن خصوصية بيانات كل منظمة.

### **المرحلة الثانية: محرك إنشاء الاختبارات والذكاء الاصطناعي**
- **الهدف:** توفير أدوات ذكية لإنشاء المحتوى التعليمي.
- **ما تم إنجازه:**
    - واجهة لإضافة الأسئلة يدوياً (MCQ، صح وخطأ، مقالي).
    - دمج نموذج **Gemini Pro** لتوليد بنوك أسئلة ذكية من الأوصاف النصية.
    - دعم رفع ملفات **PDF وWord** واستخراج النص منها تلقائياً لتحويلها لأسئلة.
    - نظام تشفير (AES-256) لحماية مفاتيح API الخاصة بالمستخدمين.

### **المرحلة الثالثة: تجربة الطالب والتصحيح التلقائي**
- **الهدف:** توفير بوابة اختبار آمنة وسهلة للطلاب.
- **ما تم إنجازه:**
    - توليد روابط اختبار فريدة وغير قابلة للتخمين لكل اختبار.
    - واجهة اختبار تفاعلية (Progress Bar، مؤقت، تنقل سلس).
    - بوابة دخول (Candidate Gate) لجمع بيانات الطالب قبل البدء.
    - نظام تصحيح خلفي (Server-side) يمنع تسريب الإجابات أو تزوير النتائج.

### **المرحلة الرابعة: لوحة التحكم والتحليلات المتقدمة**
- **الهدف:** تمكين المعلمين من اتخاذ قرارات مبنية على البيانات.
- **ما تم إنجازه:**
    - رسوم بيانية تفاعلية (Recharts) لمتوسط الدرجات وأداء الطلاب.
    - تحليل أصعب الأسئلة (Fail Rate) لتحديد فجوات التعلم.
    - نظام تصحيح ذكي للمقالات باستخدام AI مع تقديم تغذية راجعة شخصية.
    - واجهة متقدمة لإدارة فريق المساعدين وتوزيع الصلاحيات.

### **المرحلة الخامسة: إعادة التصميم والتوثيق القانوني**
- **الهدف:** تخصيص المنصة بالكامل لهوية "المهندسة أحلام".
- **ما تم إنجازه:**
    - إعادة تصميم الصفحة الرئيسية لتركز على رسالة المنصة وأدوات المهندسة أحلام.
    - إزالة كافة الروابط المرجعية للقالب الأصلي.
    - إنشاء صفحات قانونية (الخصوصية، الشروط) ومركز مساعدة متكامل باللغة العربية.

## 2. ميثاق التصميم والتقنيات المستخدمة

### **أ- فلسفة التصميم (Design Identity):**
*   **السمة البصرية:** تعتمد المنصة سمة **"الأزرق الملكي" (Royal Blue)** التي تعكس الاحترافية والهدوء والثقة، وهي مثالية للبيئات التعليمية.
*   **تجربة المستخدم (UX):** تصميم بسيط (Minimalist) يركز على المهام الأساسية، مع واجهة مستخدم متجاوبة (Responsive) تعمل بكفاءة على كافة الأجهزة (حاسوب، تابلت، موبايل).
*   **التوطين:** دعم كامل للغة العربية (RTL) مع خطوط مهنية واضحة تناسب المحتوى الأكاديمي.

### **ب- المكاتب والتقنيات المستخدمة:**

| المكتبة / التقنية | الغرض من الاستخدام |
| :--- | :--- |
| **Next.js 15 (React 19)** | الإطار البرمجي الأساسي للمنصة (App Router). |
| **Supabase** | قاعدة البيانات (PostgreSQL)، المصادقة (Auth)، والسياسات الأمنية (RLS). |
| **Google Gemini API** | توليد الأسئلة ذكياً وتصحيح المقالات وتقديم التغذية الراجعة. |
| **pdf-parse** | استخراج النصوص من ملفات PDF المرفوعة. |
| **mammoth** | تحويل ملفات Word (.docx) إلى نص خام للمعالجة. |
| **Recharts** | بناء الرسوم البيانية التفاعلية في لوحة التحليلات. |
| **Lucide React** | نظام الأيقونات المستخدم في كامل الواجهات. |
| **crypto (Node.js)** | تشفير مفاتيح API الخاصة بالمستخدمين (AES-256-CBC). |
| **Tailwind CSS** | تصميم الواجهات المتجاوبة والمتناسقة مع الهوية البصرية. |

## 3. كيفية تشغيل وتفعيل التطوير

### **أ- متطلبات التشغيل (Environment Variables):**
يجب إعداد الملف `.env` في مجلد `nextjs` بالقيم التالية:
- `NEXT_PUBLIC_SUPABASE_URL`: رابط مشروع Supabase.
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`: مفتاح Anon الخاص بـ Supabase.
- `PRIVATE_SUPABASE_SERVICE_KEY`: مفتاح Service Role (للعمليات الخلفية والتصحيح).
- `NEXT_PUBLIC_PRODUCTNAME`: "منصة أحلام".
- `ENCRYPTION_SECRET`: مفتاح سري لتشفير مفاتيح Gemini (اختياري، يستخدم Service Key كبديل).
- `ENCRYPTION_SALT`: ملح التشفير لزيادة الأمان.
- `SMTP_HOST`, `SMTP_USER`, `SMTP_PASS`: إعدادات إرسال البريد الإلكتروني (Gmail).

### **ب- تفعيل قاعدة البيانات (Migrations):**
يجب تشغيل ملفات Migration الموجودة في `supabase/migrations/` بالترتيب:
1. `rbac_and_content.sql`: الأدوار وبنية الاختبارات.
2. `phase2_schema.sql`: مواضيع الأسئلة وتشفير المفاتيح.
3. `phase3_schema.sql`: روابط الطلاب والنتائج.
4. `harden_rls.sql` و `security_hardening.sql`: حماية البيانات ومنع التسريب.
5. `persistence.sql`: حفظ إجابات الطلاب التفصيلية.
6. `notifications.sql`: نظام الإشعارات.

### **ج- تشغيل المنصة محلياً:**
```bash
cd nextjs
npm install
npm run dev
```

### **د- النشر عبر Vercel:**
1. اربط مستودع GitHub الخاص بك بمنصة Vercel.
2. اختر مجلد `nextjs` كمجلد أساسي (Root Directory).
3. أضف كافة متغيرات البيئة المذكورة أعلاه في إعدادات Vercel.
4. تأكد من تشغيل كافة الـ Migrations في Supabase قبل عملية الـ Build الأولى.

## 4. طريقة استخدام المنصة

### **1. دور المحاضر (Teacher):**
- **الإعداد:** الذهاب إلى "إعدادات المستخدم" وإضافة مفتاح Gemini API وتشفيره.
- **إدارة المحتوى:** إنشاء "مادة دراسية" ثم "بنك أسئلة".
- **الذكاء الاصطناعي:** استخدام ميزة "توليد الأسئلة" عبر وصف المادة أو رفع ملف PDF/Word.
- **الاختبارات:** إنشاء اختبار جديد، اختيار الأسئلة، ثم نسخ "رابط الطالب".
- **التصحيح:** مراجعة إجابات الطلاب في صفحة "Grading" واستخدام "التصحيح الذكي" للمقالات.

### **2. دور المساعد (Assistant):**
- يمكنه إضافة أسئلة لبنك الأسئلة ومراجعة النتائج، لكنه لا يملك صلاحية حذف الاختبارات أو المنظمات.

### **3. دور الطالب (Student):**
- الدخول عبر الرابط الفريد، إدخال الاسم والبريد الإلكتروني، البدء في الحل، والحصول على النتيجة فور الانتهاء (للأسئلة الموضوعية).

### **4. دور الأدمن (Admin):**
- مراقبة كافة المنظمات، إدارة الدعوات العالمية، والاطلاع على التحليلات الشاملة للمنصة عبر `/admin`.

## 5. الفوائد والمميزات الاستراتيجية
*   **توفير الوقت:** تقليل عبء إعداد الاختبارات بنسبة 80% بفضل أدوات الذكاء الاصطناعي.
*   **الأمان القصوى:** حماية الأسئلة والنتائج من خلال تقنيات تشفير متطورة وسياسات وصول صارمة.
*   **دقة التقييم:** نظام تصحيح ذكي يضمن العدالة حتى في الأسئلة المقالية.
*   **المرونة:** إمكانية إدارة فصول دراسية ومنظمات تعليمية متعددة من حساب واحد.

## 6. الرؤية والتطوير المستقبلي
نحن نطمح لجعل "منصة أحلام" الخيار الأول للتعليم الذكي في المنطقة:
*   **المراقبة بالذكاء الاصطناعي (AI Proctoring):** إضافة ميزات كشف الغش عبر الكاميرا والتحليل السلوكي أثناء الاختبار.
*   **التعلم التكيفي (Adaptive Learning):** نظام يغير صعوبة الأسئلة تلقائياً بناءً على مستوى أداء الطالب.
*   **تطبيقات الموبايل:** إطلاق تطبيقات أصلية (Native Apps) لنظامي iOS و Android لتسهيل الوصول.
*   **التكامل مع LMS:** ربط المنصة مع أنظمة عالمية مثل Moodle و Google Classroom.
