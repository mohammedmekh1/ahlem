# دليل إعداد وتشغيل المنصة (للموظفين والمطورين)

هذا الدليل مخصص لشرح كيفية استلام المشروع، إعداده، تشغيله، ونشره للإنتاج من الصفر. لقد تم تجهيز كافة الأدوات والبنية التحتية داخل المستودع لتسهيل هذه العملية.

---

## 🛠️ أولاً: المتطلبات التقنية
يجب التأكد من وجود الأدوات التالية مثبتة على جهاز المطور:
1.  **Node.js (v20+):** المحرك الأساسي لتشغيل الكود.
2.  **Yarn / NPM:** لإدارة المكتبات.
3.  **Supabase CLI:** للتحكم في قاعدة البيانات (ثبته عبر `npm install -g supabase`).
4.  **Git:** لسحب ورفع الكود.

---

## 📦 ثانياً: الخطوات التنفيذية (من الصفر)

### 1. تجهيز الكود المصدري
*   قم بعمل `clone` للمستودع.
*   ادخل لمجلد `nextjs` وقم بتثبيت المكتبات:
    ```bash
    cd nextjs
    yarn install
    ```

### 2. ربط قاعدة البيانات (Supabase)
*   أنشئ مشروعاً جديداً على [Supabase](https://supabase.com).
*   في الجهاز الطرفي (Terminal)، نفذ الأوامر التالية بالترتيب:
    ```bash
    # تسجيل الدخول
    npx supabase login

    # ربط المشروع المحلي بمشروع سوبابيس (استخدم معرف المشروع من الإعدادات)
    npx supabase link --project-ref [YOUR_PROJECT_ID]

    # رفع الهيكل البرمجي (جداول المنظمات، الإدارة، الصلاحيات)
    npx supabase db push
    ```

### 3. إعداد المتغيرات البيئية (المفاتيح السرية)
*   انسخ ملف `.env.template` إلى `.env.local`:
    ```bash
    cp .env.template .env.local
    ```
*   قم بتعبئة الملف بالقيم التالية من لوحة تحكم Supabase:
    *   `NEXT_PUBLIC_SUPABASE_URL`
    *   `NEXT_PUBLIC_SUPABASE_ANON_KEY`
    *   `PRIVATE_SUPABASE_SERVICE_KEY`
    *   **هام:** تأكد من أن `NEXT_PUBLIC_THEME=theme-sass3` لظهور اللون الأزرق الملكي.

## 🚀 ثالثاً: التشغيل والاختبار

### 1. وضع التطوير
لتشغيل المشروع ومعاينته محلياً:
```bash
npm run dev
```

### 2. الاختبارات الآلية (يجب تشغيلها قبل أي رفع للكود)
*   **اختبارات المنطق:** `npm test`
*   **اختبارات الواجهة:** `npm run test:e2e`

---

## 🌐 رابعاً: النشر للإنتاج (Deployment)

لقد تم إعداد المشروع ليكون متوافقاً تماماً مع **Vercel**:
1. ارفع الكود إلى مستودع GitHub (خاص).
2. اربط المستودع بـ Vercel.
3. **نقطة حاسمة:** انقل كافة القيم من ملف `.env.local` إلى إعدادات Vercel (Environment Variables).
4. اضغط على **Deploy**. سيقوم النظام ببناء المشروع وتفعيله تلقائياً.

## ⚙️ خامساً: تفعيل الأنظمة المتقدمة (الأتمتة والمراقبة)

لقد جهزت لك البنية التحتية، وكل ما عليك هو تفعيلها عبر "GitHub Secrets":
1. اذهب لمستودعك على GitHub > Settings > Secrets and variables > Actions.
2. أضف سر جديد باسم **`SUPABASE_DB_URL`** (رابط الاتصال المباشر بقاعدة البيانات) لتفعيل **النسخ الاحتياطي التلقائي اليومي**.
3. أضف سر باسم **`SENTRY_DSN`** (من موقع Sentry) لتفعيل **مراقبة الأخطاء**.
4. بمجرد الرفع، سيقوم **GitHub Actions** بفحص الكود (CI) للتأكد من جودته.

---

## 🛡️ خامساً: إدارة المنصة والصلاحيات (Admin & Roles)
المنصة مجهزة بنظام تحكم متقدم:
1.  **تفعيل الآدمن:** بعد تسجيل أول حساب، اذهب لجدول `profiles` في Supabase وقم بتغيير `is_admin` إلى `true`.
2.  **لوحة الإدارة:** يمكن الوصول إليها عبر الرابط `/admin`. تتيح لك رؤية إحصائيات المنصة (المشتركون حالياً 0).
3.  **تعدد المنظمات:** يمكن لكل مستخدم إنشاء منظمة (Organization) ودعوة أعضاء. يتم عزل البيانات تلقائياً بناءً على `org_id`.

## 🤖 سادساً: نظام الذكاء الاصطناعي (AI System)
*   **الموقع:** موجود في صفحة `/app/ai`.
*   **الوظيفة:** يتيح للمستخدمين كتابة الأسئلة والحصول على تصحيح.
*   **ملاحظة تقنية:** الكود مهيأ للربط مع OpenAI API؛ يحتاج المطور فقط لإضافة مفتاح الـ API في ملف الإعدادات وربط الدالة البرمجية.
