# قالب SaaS المتكامل (Next.js 15 + Supabase)

دليل شامل من الصفر إلى الاحتراف لتشغيل واستخدام القالب.

## 🚀 المميزات
*   **نظام التحقق:** تسجيل الدخول، التحقق الثنائي (MFA)، وربط الحسابات الاجتماعية.
*   **تعدد المنظمات (Multi-tenancy):** دعم كامل للفرق والمنظمات مع حماية البيانات.
*   **لوحة تحكم تفاعلية:** رسوم بيانية لعرض النشاط باستخدام Recharts.
*   **نظام الاختبارات:** اختبارات برمجية (Vitest) واختبارات شاملة (Playwright).
*   **المراقبة:** دمج نظام Sentry لتتبع الأخطاء في الإنتاج.

---

## 🛠️ المتطلبات الأساسية
1.  **Node.js (الإصدار 20 أو أحدث).**
2.  **Yarn أو NPM.**
3.  **حساب على [Supabase](https://supabase.com).**

---

## 📦 الخطوة 1: إعداد البيئة المحلية
1.  **تحميل الكود:**
    ```bash
    git clone [رابط_المستودع]
    cd [اسم_المجلد]
    ```
2.  **تثبيت المكتبات:**
    ```bash
    cd nextjs
    npm install
    ```

---

## ☁️ الخطوة 2: إعداد Supabase
1.  **إنشاء مشروع جديد:** اذهب إلى لوحة تحكم Supabase وأنشئ مشروعاً.
2.  **ربط المشروع:**
    ```bash
    npx supabase login
    npx supabase link --project-ref [معرف_مشروعك]
    ```
3.  **رفع قاعدة البيانات:**
    ```bash
    npx supabase migrations up --linked
    ```

---

## ⚙️ الخطوة 3: الإعدادات (Environment Variables)
1.  في مجلد `nextjs` قم بعمل نسخة من الملف القالب:
    ```bash
    cp .env.template .env.local
    ```
2.  افتح الملف `.env.local` وضع القيم التالية:
    *   `NEXT_PUBLIC_SUPABASE_URL`: رابط المشروع.
    *   `NEXT_PUBLIC_SUPABASE_ANON_KEY`: المفتاح العام (Anon Key).
    *   `PRIVATE_SUPABASE_SERVICE_KEY`: مفتاح الخدمة (Service Role).

---

## 🚀 الخطوة 4: التشغيل والاستخدام
1.  **تشغيل التطبيق:**
    ```bash
    npm run dev
    ```
2.  افتح `http://localhost:3000`.
3.  **التسجيل:** أنشئ حساباً جديداً، وستنتقل تلقائياً إلى لوحة التحكم التفاعلية.

---

## 🧪 الاختبارات والمراقبة
*   **تشغيل الاختبارات البرمجية:** `npm test`
*   **تشغيل اختبارات المتصفح:** `npm run test:e2e`
*   **المراقبة:** بمجرد وضع `SENTRY_DSN` في إعدادات البيئة، سيبدأ النظام بتسجيل الأخطاء تلقائياً.

---

## 🌐 النشر (Production)
الخيار الأفضل هو استخدام **Vercel**:
1. ارفع الكود إلى GitHub.
2. اربط المستودع بـ Vercel.
3. انقل القيم الموجودة في `.env.local` إلى إعدادات البيئة في Vercel.
